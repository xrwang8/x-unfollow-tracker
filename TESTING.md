# 实机测试指南

验证扩展所有功能是否正常工作，包括 API 降级方案。

## 测试前准备

### 1. 构建扩展
```bash
cd /Users/xrwang/go/src/github.com/xrwang8/x-unfollow-tracker
npm run build
```

### 2. 加载扩展到 Chrome
1. 打开 Chrome，访问 `chrome://extensions/`
2. 右上角开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择目录：`.output/chrome-mv3`
5. 确认扩展加载成功（工具栏出现蓝色图标）

### 3. 登录 X
在新标签页打开 https://x.com 并登录你的账号

### ⚠️ 降级方案测试（高级）

#### 测试目标
验证 API 失效时自动切换到 DOM 抓取。

#### 方法 1：模拟 API 失效（推荐）

1. 打开扩展源码 `lib/x-api.ts`
2. 临时修改 bearer token（让 API 返回 401）：
   ```typescript
   const X_BEARER = "Bearer INVALID_TOKEN_FOR_TESTING";
   ```
3. 重新构建并重载扩展：
   ```bash
   npm run build
   # 在 chrome://extensions/ 点击扩展的"重新加载"按钮
   ```
4. 打开 https://x.com/你的用户名/followers 页面
5. 点击扩展图标 → "检查关注者"
6. **预期行为**：
   - [ ] 进度提示显示 `[页面滚动] 已拉取 XXX 人...`
   - [ ] 页面自动滚动（可见）
   - [ ] 完成后显示"快照已保存：XXX 人 (使用备用方案)"
   - [ ] Console 显示警告："API 失效，降级到 DOM 抓取"

**预期耗时**：500 人 ≈ 12-15 分钟（慢）

#### 方法 2：直接测试 DOM 抓取

暂时不推荐，因为需要修改代码强制使用 DOM 模式。

## 调试技巧

### 1. 查看 Console 日志
- 在 `chrome://extensions/` 点击扩展的"检查视图：popup.html"
- 打开 DevTools Console
- 查看错误日志和 API 响应

### 2. 查看存储数据
在 Console 中执行：
```javascript
chrome.storage.local.get(null, (data) => console.log(data));
```

### 3. 清空所有数据
```javascript
chrome.storage.local.clear(() => console.log('cleared'));
```

### 4. 模拟网络慢速
- DevTools → Network → Throttling → Slow 3G
- 测试 API 超时处理

## 已知问题排查

### 问题 1：点击按钮无反应
**可能原因**：
- 未登录 x.com
- popup DevTools 未打开，看不到错误

**解决**：
- 检查 x.com 登录状态
- 打开 popup DevTools 查看 Console 错误

### 问题 2：进度卡在某个数字不动
**可能原因**：
- API rate limit（429）
- 网络问题

**解决**：
- 等待 1-2 分钟
- 检查网络连接

### 问题 3：DOM 抓取失败
**可能原因**：
- 不在正确的页面（需要在 followers/following 页面）
- X 改了 HTML 结构

**解决**：
- 确保在 `x.com/用户名/followers` 页面
- 检查 DevTools Console 错误
